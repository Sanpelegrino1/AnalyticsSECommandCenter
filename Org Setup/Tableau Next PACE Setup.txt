BUILD 2: Build Data Streams, Data Lake Objects,
and Data Models

TIME NEEDED: 30 to 45 minutes

 TERMINOLOGY (want more detail? Click here)

Data Stream                      Brings in the raw data.

Data Lake Object (DLO)           Stores the raw data as it arrives.

Data Model (DMO)                 Transforms and organizes the raw data into a structured and usable
                                 format for analysis and activation.

Semantic Data Model (SDM)        Data Cloud layer where you can define, govern, and query all of your
                                 business metrics. Semantic models provide a consistent way to
                                 interpret data across Salesforce by mapping your data to familiar
                                 business terms and standardized business logic. Tableau Semantics
                                 also uses a Query Generator to leverage the data to answer key
                                 business questions by sending customizable semantic queries based
                                 on your semantic models.

                 Things to Keep in Mind

If you miss a step (like adding all of your fields), you can delete your Data Stream and start over.
You will notice this as you start the process of building your DMO and only show the primary key
listed in the fields. If this happens, you will need to first delete the in-process DMO from the list
and then you can delete the Data Stream and start over.

If you were updating data from a stream, you WILL want to pull in only the fields that you need as
this impacts performance and DC credits. Since we are not setting up any data to update, I wasn't
focused on it but you can choose to only pull in relevant fields.

For this build we are going to connect to our Heroku PostgreSQL data and
supplement with additional data so you can see how simple it is to customize your
data without recreating it. You can also pull in all custom tables if that is your
preference...generally, the process will be the same. We will also update records in
Salesforce (later build) so you can create an Action framework in your dashboards.

BUILD 3: Build Your SDM with Calculations,
Parameters, Metrics, Descriptions, and More

TIME NEEDED: 2+ hours

A Semantic Model is a container of the entities that define your business terms. The
Semantic Model is composed of a grouping of semantic definitions tailored for a
specific analytical use case. Semantic Models consist of one or more data objects.
With Semantic Models, you can create flexible joins across more than four objects
and enrich your analysis with Calculated Dimensions and Measures.

You can think of it as a translation layer between the raw, messy data in your
spreadsheet or database, where you store fields with text and numbers, and the
business language we all speak, such as Ł Metrics and Dimensions, or Revenue and
Regions. It serves as a trusted place where you can ask business questions of your
data such as "what was my Revenue across Regions during the Last Year?"

Concierge operates within the context of what you are currently looking at in
Tableau Next. If you're viewing a specific dashboard, it will use the semantic model
associated with that dashboard to understand your questions and provide answers.
This means you can have multiple Tableau semantic models (layers) in your
Salesforce org and still enable Concierge to answer questions correctly. The key is
that Concierge references the semantic model tied to the specific metric or
dashboard you are viewing to ensure accuracy

You can build the Semantic Model from within Data Cloud OR Tableau Next. Since
we are learning about Tableau Next, let's add it in the Tableau Next app. Regardless
of where you start your SDM build, you will be able to edit it in either place;
however, to do INLINE edits, you MUST create the SDM in Tableau Next.

ŁGet Deeper Into DATA CLOUD

Data Stream:
A data stream represents the connection and flow of data from a source system into
Data Cloud. It defines how data is ingested, including the source, frequency, and initial
processing rules. Think of it as the pipeline through which raw data enters the system.

Data Lake Objects (DLOs):
Once data flows through a data stream, it is stored in its raw, original format within data

                                                                                                                             42
Tableau Next ICE Learning Guide

lake objects. DLOs are essentially storage containers within the data lake, holding the
ingested data as-is, without imposing a strict schema. This allows for flexibility and the
ability to store various data types and formats. DLOs are the first point where the raw
data resides within the data cloud.

Data Model Objects (DMOs):
Data Model Objects represent the harmonized and standardized view of the data. After
data is ingested into DLOs, it is then mapped to DMOs. This mapping process involves
defining the relationships between different data points, standardizing data formats,
and applying business rules to create a unified and structured data model. DMOs are
the foundation for segmentation, activation, and analytical processes within the data
cloud, providing a clean and consistent view of customer data.

Semantic Data Model (SDMs):
The Semantic Layer creates a business-friendly, abstract layer that defines the meaning
and relationships between data elements, translating complex technical data
structures into understandable business terms and metrics for human users and
machines. It focuses on the meaning and context of data, rather than just its technical
structure, to enable better understanding, consistent reporting, and more informed
decision-making across an organization.

Customer 360 Data Model:
Data Cloud's standard canonical data model. Data ingested into Data Cloud is mapped
to DMOs found in the Customer 360 Data Model.
